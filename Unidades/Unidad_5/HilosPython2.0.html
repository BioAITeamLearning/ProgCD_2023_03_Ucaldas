¬øC√≥mo utilizar los hilos?#

La forma m√°s sencilla de usar un subproceso es instanciarlo con una funci√≥n de destino y luego llamar al m√©todo start() para que comience a funcionar.

El subprocesamiento del m√≥dulo de Python tiene el m√©todo Thread():

class threading.Thread(group=None, target=None, name=None, args=(), kwargs={})

  • group: Este es el valor del grupo que debe ser Ninguno; esto est√° reservado para futuras implementaciones

  • target: esta es la funci√≥n que se ejecutar√° cuando inicie una actividad de subproceso

  • name: Este es el nombre del hilo; por defecto, se le asigna un nombre √∫nico de la forma Thread-N

  • args: esta es la tupla de argumentos que se pasar√°n a un destino

  • kwargs: este es el diccionario de argumentos de palabras clave que se utilizar√°n para la funci√≥n de destino

¬øC√≥mo implemento un hilo en una subclase?#

  • Definir una nueva subclase de la clase Thread

  • Llamar al constructor de la clase Thread es obligatorio; us√°ndolo, podemos redefinir algunas propiedades del hilo como el nombre del hilo.

  • Sobrecargue el m√©todo init_(self [,args]) para agregar argumentos adicionales

  • Luego, debo sobrecargar el m√©todo run(self [,args]) para implementar lo que debe hacer el subproceso cuando se inicia

  • En el programa principal, creamos varios objetos del tipo myThread; la ejecuci√≥n del hilo comienza cuando se llama al m√©todo start().

  • El subproceso se coloca en el estado activo de la llamada a start() y permanece all√≠ hasta que finaliza el m√©todo run() o lanza una excepci√≥n no controlada.

  • El programa finaliza cuando todos los subprocesos finalizan.

Caso 1#

import threading
import time

class myThread (threading.Thread):
    def __init__(self, threadID, name, counter):
        threading.Thread.__init__(self)
        self.threadID = threadID
        self.name = name
        self.counter = counter
    def run(self):
        print ("Iniciando " + self.name)
        print_time(self.name, 5,self.counter)
        print ("Finalizando " + self.name)

def print_time(threadName, delay, counter):
    while counter:
        time.sleep(delay)
        print ("%s: %s\n" %(threadName, time.strftime("%H:%M:%S", time.localtime(time.time())))) # Thread-#: HH:MM:SS
        counter -= 1

# Crear hilos
thread1 = myThread(1, "Thread-1", 1) # Objetos del tipo myThread # Hilo que muestra la hora cada 1 segundo
thread2 = myThread(2, "Thread-2", 3) # Objetos del tipo myThread # Hilo que muestra la hora cada 4 segundos

¬øHay que tomar precauciones al utilizar los hilos?#

  • Mientras el programa principal ha llegado al final, el hilo contin√∫a imprimiendo su mensaje cada dos segundos.

  • Este ejemplo demuestra qu√© son los subprocesos: una subtarea que hace algo en un proceso principal.

Un punto clave a tener en cuenta al usar subprocesos es que siempre debe asegurarse de nunca dejar ning√∫n subproceso ejecut√°ndose en segundo plano. Esta es una programaci√≥n muy mala y puede causarle todo tipo de problemas cuando trabaja en aplicaciones m√°s grandes.

from threading import Thread
from time import sleep

class CookBook(Thread):
    def __init__(self):
        Thread.__init__(self)
        self.message = "Hilo Trabajando!\n"

    def print_message(self):
        print (self.message)

    def run(self):
        print ("Hilo Inicia\n")
        x=0
        while (x < 3):
            self.print_message()
            sleep(2)
            x += 1
        print ("Hilo Finaliza\n")

print('\nPrograma principal inicia.\n')
hello_Python = CookBook()
hello_Python.start()
print('\nPrograma principal finaliza.\n')

# Librerias que utilizaremos
import threading   # Manejo de hilos
import numpy as np # Manejo de matrices
import time        # Manejo de tiempos

# Definir las 3 matrices de 4x10, con n√∫meros enteros entre 0 y 10
matrix1 = np.round(10*np.random.rand(4, 10)).astype(np.int8)
matrix2 = np.round(10*np.random.rand(4, 10)).astype(np.int8)
matrix3 = np.round(10*np.random.rand(4, 10)).astype(np.int8)

Bloqueos - Lock#

  • Un Lock es un objeto que se utiliza para bloquear el acceso a una secci√≥n cr√≠tica del c√≥digo, es decir, una secci√≥n que s√≥lo puede ser ejecutada por un hilo a la vez.

  • Si creamos un objeto de clase Lock los hilos podr√°n invocar a sus m√©todos acquire y release.

  • El lock es como una puerta que inicialmente est√° abierta. Cuando un hilo llega a la puerta, debe invocar el m√©todo acquire para pasar por ella y cerrarla. En ese momento, el hilo tiene acceso al recurso compartido que se est√° regulando detr√°s de la puerta.

  • Si otro hilo llega a la puerta y hace acquire, pero la puerta ya est√° cerrada por otro hilo, este segundo hilo se queda esperando en la puerta.

  • Cuando el primer hilo termina de usar el recurso compartido, debe abrir la puerta invocando el m√©todo release. Esto abrir√° la puerta y permitir√° que uno de los hilos que estaba esperando pase y cierre la puerta.

  • No controla qui√©n hace release, de forma que cualquier otro hilo podr√≠a hacerlo.

import threading

Sem√°foros - Semaphore#

  • Los sem√°foros permiten regular el acceso a un recurso compartido por un n√∫mero preestablecido de hilos.

  • Cada hilo que desee acceder al recurso debe llamar al m√©todo acquire y, cuando haya terminado, debe llamar al m√©todo release.

  • La clase Semaphore cumple con el protocolo de gesti√≥n de contexto, lo que significa que se puede utilizar con la construcci√≥n with para garantizar que el recurso compartido se libere correctamente, incluso en caso de errores o excepciones.

import threading
import time

def imprimir_mensaje(num, semaforo):
    semaforo.acquire()
    print("Hola, soy el hilo", num)
    time.sleep(3)
    semaforo.release()

Barreras - Barrier#

  • Las barreras son √∫tiles cuando un grupo de hilos necesita esperar mutuamente unos por otros.

  • Para utilizarlas, creamos una instancia de la clase Barrier con el n√∫mero de hilos que deben quedar en espera en la barrera.

  • Cuando un hilo llega a la barrera, llama al m√©todo wait y espera a que los dem√°s hilos tambi√©n lo hagan. Cuando todos los hilos han llamado a wait, la barrera se abre y todos los hilos contin√∫an su ejecuci√≥n.

  • Las barreras son especialmente √∫tiles cuando los hilos tienen funcionalidades diferentes y necesitan esperarse mutuamente hasta que est√©n preparados.

Vamos a establecer una barrera de forma que nadie pueda despedirse si no ha saludado todo el mundo (es decir, los hilos).

import threading
import time
import random

def worker(num, barrier):
    sleep(random.random() * 2)
    print(f"worker {num} saluda...")
    barrier.wait()  # esperar a que todos los hilos lleguen a la barrera
    sleep(random.random() * 2)
    print(f"worker {num} se despide...")

# crear barrera para 3 hilos
barrier = threading.Barrier(3)

# crear e iniciar hilos
thread1 = threading.Thread(target=worker, args=(1, barrier))
thread2 = threading.Thread(target=worker, args=(2, barrier))
thread3 = threading.Thread(target=worker, args=(3, barrier))

thread1.start()
thread2.start()
thread3.start()
